# Software Requirements Specification: TV Assassin

## 1. Executive Summary
TV Assassin is a minimalist Android mobile application that enables users to send IR shutdown signals to multiple television brands using a single button interface. Designed for technical enthusiasts, the app prioritizes speed and simplicity, leveraging device IR blasters to broadcast shutdown commands with minimal user interaction. This document outlines functional, technical, and UX requirements compliant with F-Droid open-source standards.

## 2. Project Overview
**Problem Statement**: Existing universal remote apps require complex setup and lack one-touch TV shutdown capabilities.  
**Solution**: Lightweight app providing immediate IR signal broadcasting upon button press.  
**Key Innovations**:  
- Sub-500ms execution  
- Zero-configuration operation  

## 3. Scope and Objectives
**In Scope**:  
- IR blaster detection/validation  
- Single-command multi-signal broadcasting  
- Theme switching (light/dark)  

**Out of Scope**:  
- TV brand detection/recognition  
- Remote control functionality beyond shutdown  
- iOS/Web implementations  

## 4. Functional Requirements

### Core Functions
**FR-001**: IR Blaster Detection *(Must Have)*  
The app shall automatically detect available IR blaster hardware on launch and display:  
1. Success confirmation icon  
2. Error message if unavailable ("No IR blaster detected")  

**FR-002**: Signal Transmission *(Must Have)*  
When the "Kill TVs" button is pressed:  
1. Broadcast all shutdown codes sequentially (minimum 15 brands)  
2. Disable button during transmission (if >500ms)  
3. Show transmission complete haptic feedback  

**FR-003**: Theme Management *(Should Have)*  
1. Default to system theme  
2. Toggle between light/dark/system via app bar button  
3. Persist user selection across sessions  

### Edge Cases
**FR-004**: No IR Hardware *(Could Have)*  
Display persistent warning banner if IR hardware becomes unavailable during session  

## 5. Non-Functional Requirements
**NFR-001**: Performance *(Must Have)*  
- App launch <300ms (cold start)  
- APK size ≤50MB  
- Signal transmission complete in ≤500ms  

**NFR-002**: Reliability *(Must Have)*  
- 99% successful IR command transmission on supported devices  
- Zero unhandled exceptions  

**NFR-003**: Usability *(Should Have)*  
- "Kill TVs" button ≥20% of screen height in portrait  
- Contrast ratio ≥4.5:1 for accessibility  

## 6. System Architecture
**Recommendations**:  
- Flutter framework (Dart)  
- MVVM pattern with:  
  - `IRService`: Hardware abstraction  
  - `CommandRepository`: Signal code storage  
  - `ThemeManager`: UI state handling  
- SharedPreferences for theme persistence  

## 7. User Interface Requirements
**UI-001**: Main Screen *(Must Have)*  
- Full-screen centered button with:  
  - Skull icon (Material Design)  
  - "Kill TVs" label  
  - Press animation  

**UI-002**: App Bar *(Should Have)*  
- Right-aligned theme toggle (sun/moon icons)  
- No navigation elements  

## 8. Data Requirements
**DR-001**: Signal Database *(Must Have)*  
- Hardcoded Map<String, List<int>> of IR codes  
- Structure: `{"BrandA": [0x..., ...], ...}`  
- Minimum 15 major brands covered  

## 9. Security and Privacy Requirements
**SEC-001**: Permissions *(Must Have)*  
- Single declared permission: `android.hardware.consumerir`  

**SEC-002**: Data Handling *(Must Have)*  
- No analytics collection  
- No network requests  

## 10. Integration Requirements
**INT-001**: Android API *(Must Have)*  
- ConsumerIrManager API access  
- Target SDK 34 (Android 14)  

## 11. Testing Requirements
**TEST-001**: Unit Tests *(Should Have)*  
- 100% coverage for IR command sequencing  
- Mock testing of hardware failures  

**TEST-002**: Device Testing *(Must Have)*  
- Verify on one or more IR-capable devices  
- Validate theme persistence  

## 12. Deployment and Operations
**DEP-001**: Build Process *(Must Have)*  
- Single-command Flutter build  
- F-Droid compliant APK signing  

**DEP-002**: Maintenance *(Could Have)*  
- Documentation for code updates:  
  - Adding new IR codes  
  - Theme modifications
